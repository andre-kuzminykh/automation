Почему поведение важнее абстракции

Абстрактный запрос

"Сделай диагностику ИИ-зрелости"

Поведенческий запрос

"Если пользователь нажал кнопку старт,
покажи первый вопрос, сохрани ответ в state,
и переведи к следующему шагу."

ИИ лучше реализует то, что описано как система реакций

Два режима Code Node

For Each Item

  • — Запускается на каждый item отдельно
  • — Работает через $json
  • — Подходит для локальной обработки
1
2
3
(отдельно)

Once for All Items

  • — Запускается один раз на весь массив
  • — Работает через $input.all()
  • — Подходит для работы со всем набором
1
2
3
(единожды)

Выбор режима меняет поведение всей ноды

Когда использовать какой режим

Нужно видеть все элементы (сравнивать)?
НЕТ

For Each Item

  • Форматирование
  • Переименование полей
  • Преобразование дат
  • Локальные вычисления
ДА

Once for All Items

  • Суммы (Агрегация)
  • Сортировка
  • Дедупликация
  • Группировка

Почему Code Node так силён

Предсказуемый результат

Никаких галлюцинаций, работает строго по логике.

Нет затрат на токены

Бесплатное исполнение без вызовов внешних API.

Быстрее ИИ-агентов

Миллисекунды вместо секунд ожидания ответа от LLM.

Большие объёмы данных

Обработка тысяч записей без проблем с контекстом.

Code Node делает пайплайн стабильнее, дешевле и надёжнее

Итог лекции

Vibe-coding — это смена мышления, а не просто новый синтаксис.

Хороший результат рождается из требований, поведения, тестов и итераций.

Code Node в n8n даёт контроль и надёжность там, где ИИ может ошибаться.

Будущее за теми, кто умеет мыслить как инженер и формулировать намерения.

Не обязательно уметь кодить.
Важно уметь думать как инженер.
